# Backend Notes

A collection of backend engineering notes and practical references for common systems, data stores, and architecture patterns.

## Overview

This repository is organized into topic folders and notes for core backend systems:

- `Kafka/` - Apache Kafka concepts, architecture, and internal design
- `Redis/` - Redis fundamentals, data types, commands, and Pub/Sub

The goal is to provide quick, readable backend references for engineers learning or reviewing infrastructure, messaging, caching, and distributed system concepts.

## Topics Covered

### Messaging and Event Streaming

- Kafka architecture and cluster behavior
- Topics, partitions, brokers, and replication
- Producers, consumers, and consumer groups
- Ordering, offsets, and fault tolerance
- Practical Kafka mental models and internals

### In-Memory Data Stores

- Redis architecture and use cases
- Redis data types: strings, hashes, lists, sets, sorted sets
- Key management, expiration, and persistence
- Common Redis commands and patterns
- Pub/Sub messaging in Redis

### System Design and Backend Patterns

This repository is also intended to support broader backend engineering learning:

- distributed system principles
- scaling and partitioning concepts
- caching and storage tradeoffs
- pub/sub and event-driven design
- reliability and availability patterns
